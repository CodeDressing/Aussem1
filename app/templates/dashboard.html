Source Governance Rules

Aussem1 must be explicit about what the platform knows, what it does not know, and which source type is required for each claim.

Strict Source Mode

Public Records Can Support

Parcel identity, block/lot, tax assessment, property tax context, recorded-document references, sale-history references, ownership references, GIS context, and MOD-IV context where source-backed.

Public Records Cannot Prove Alone

Active listing status, under-contract status, current listing price, days on market, showing availability, broker confirmation, or current MLS status.

Future Authorized Feeds Required

MLS / RESO, IDX, broker-authorized listing feeds, valuation models, comparable datasets, school APIs, flood/risk layers, and persistence will be connected as separate source-governed systems.
